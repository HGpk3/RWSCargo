# RWSCargo: стек и инфраструктура

## Принятый стек

Текущая версия проекта:

- Node.js LTS;
- Astro 5;
- TypeScript;
- Docker;
- PostgreSQL 17 и Redis 8 в optional Docker profile `infra`.

Пакеты:

- `astro`;
- `@astrojs/check`;
- `typescript`.

## Команды

Установка зависимостей:

```bash
npm install
```

Dev-сервер:

```bash
npm run dev
```

Production build:

```bash
npm run build
```

Preview:

```bash
npm run preview
```

Docker:

```bash
docker compose up -d --build
```

Сайт доступен на:

```text
http://localhost:4321
```

Запуск с PostgreSQL и Redis:

```bash
docker compose --profile infra up -d --build
```

## Текущее решение по базе данных

База данных пока не используется в бизнес-логике.

Причина: первая версия может работать как быстрый SEO-сайт с формой заявки. Хранение заявок можно решить позже после выбора канала обработки.

PostgreSQL понадобится, если появятся:

- хранение заявок;
- статусы обработки;
- админка;
- история коммуникаций;
- расчетные сценарии;
- интеграции с CRM.

Redis понадобится, если появятся:

- очереди уведомлений;
- rate limiting;
- фоновые задачи;
- кэширование внешних API;
- временные токены/сессии.

## Заявки и коммуникации

Telegram нельзя считать единственным надежным каналом для России.

Кандидаты на следующую проработку:

- форма на сайте + email;
- форма на сайте + CRM;
- форма + резервная отправка в VK;
- WhatsApp/телефон как дополнительный канал;
- VK-бот как более устойчивый вариант для части аудитории;
- Telegram только как дополнительный, а не единственный канал.

Решение по заявкам нужно принимать отдельно, когда будет понятен процесс друга: кто принимает заявки, где удобно отвечать, нужна ли история и статусы.

## Docker

Текущий `docker-compose.yml`:

- `web` - основной сайт на порту `4321`;
- `postgres` - optional profile `infra`;
- `redis` - optional profile `infra`.

Docker уже используется для локальной проверки production-сборки.
