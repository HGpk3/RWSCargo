# RWSCargo: SEO-стратегия

## Главный фокус

Продвигаем не только бренд RWSCargo, а коммерческие запросы вокруг импорта из Китая для бизнеса.

Основной кластер:

- доставка грузов из Китая в Россию;
- доставка из Китая в белую;
- импорт из Китая для бизнеса;
- коммерческие грузы из Китая;
- контейнерные перевозки из Китая;
- сборные грузы из Китая;
- проверка поставщика в Китае;
- контроль качества товара в Китае;
- доставка из Китая с документами.

## Текущая SEO-база

Уже реализовано:

- `title`: "Доставка грузов из Китая в Россию в белую - RWSCargo";
- meta description;
- canonical URL;
- Open Graph;
- Twitter meta;
- `robots.txt`;
- `sitemap.xml`;
- JSON-LD `Organization`;
- JSON-LD `Service`;
- JSON-LD `FAQPage`;
- один H1;
- несколько H2 под смысловые кластеры.

## Правила для текста

- Писать для человека, не для набора ключей.
- Ключи использовать естественно: "доставка из Китая", "в белую", "коммерческие грузы", "документы", "поставщик", "контроль".
- Не делать видимый текст внутренним SEO-комментарием. Формулировки вроде "страница должна ранжироваться" и "вопросы помогают SEO" заменить перед финалом на клиентский язык.
- Не обещать серые схемы, невозможные сроки и универсальность без ограничений.
- Больше конкретики про документы, маршрут, этапы, ответственность.

## H1/H2

Текущий H1:

```text
Белый импорт из Китая
```

Он сильный для брендинга и первого экрана, но перед финальным SEO-аудитом можно протестировать вариант с более прямым ключом:

```text
Доставка грузов из Китая в Россию в белую
```

Текущие H2 в целом закрывают услуги, грузы, процесс, FAQ и заявку. Два H2 нужно переформулировать перед релизом:

- "Страница должна ранжироваться не только по бренду";
- "Вопросы, которые помогают и клиенту, и SEO".

## Будущая структура страниц

Если сайт будет расти, главную не нужно перегружать всеми ключами. Лучше вынести отдельные посадочные:

- `/dostavka-iz-kitaya/`;
- `/dostavka-iz-kitaya-v-beluyu/`;
- `/konteynernye-perevozki-iz-kitaya/`;
- `/sbornye-gruzy-iz-kitaya/`;
- `/proverka-postavshchika-v-kitae/`;
- `/kontrol-kachestva-v-kitae/`.

На первой версии можно остаться с одной сильной главной, но готовить структуру под расширение.

## Факты для уточнения

Перед финальным наполнением нужны реальные данные:

- юридическое название компании;
- город/география работы;
- склады и партнеры в Китае;
- типы грузов, с которыми реально работают;
- ограничения по грузам;
- средние сроки по маршрутам;
- документы, которые могут оформлять;
- опыт, кейсы, цифры, отзывы.

Без подтверждения эти факты не выдумывать.
