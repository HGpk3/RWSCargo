# RWSCargo: технические правила

## Текущий стек

Основной стек проекта:

- Node.js LTS;
- Astro 5;
- TypeScript;
- CSS без тяжелого UI-фреймворка;
- Docker;
- PostgreSQL и Redis как optional profile `infra`.

Проект сейчас статический: одна Astro-страница, точечный inline-скрипт для интерактива hero, глобальные стили в `src/styles/global.css`.

## Почему Astro

- Сайт в первую очередь коммерческий и контентный.
- Нужна высокая скорость загрузки и хороший SEO-фундамент.
- Интерактивность пока точечная, без необходимости в полноценном SPA.
- Проще держать код быстрым и понятным.

Если позже появятся личный кабинет, CRM, сложная админка или много динамических сценариев, можно вынести backend на Node.js и подключить PostgreSQL/Redis по реальной необходимости.

## Структура проекта

Текущая структура:

```text
src/
  pages/
    index.astro
  styles/
    global.css
public/
  images/
  robots.txt
  sitemap.xml
docs/
  references/
scripts/
  run-astro.mjs
```

Дополнительные папки `components/`, `layouts/`, `data/` стоит добавлять только когда файл `index.astro` станет тяжело поддерживать.

## CSS

- Токены цветов, радиусов, ширины и теней лежат в `:root`.
- Не плодить одноразовые классы без необходимости.
- Не строить страницу из вложенных карточек.
- Для фиксированных интерфейсных элементов задавать стабильные размеры.
- Проверять, что русский текст не ломает кнопки и карточки.

## Интерактив hero

Текущий интерактив находится в `src/pages/index.astro` и `src/styles/global.css`.

Правила:

- карточки внутри route-board остаются `button`, а не `div`;
- фокус-стейты обязательны;
- hover не должен быть просто декоративным, он должен намекать на интерактивность;
- click раскрывает смысл этапа;
- `.cursor-ghost` - вспомогательная подсказка, она не должна перекрывать важный текст;
- при активной карточке подсказка курсора скрывается.

## SEO-техника

В проекте уже есть:

- `title` и `description`;
- canonical;
- Open Graph;
- Twitter meta;
- `robots.txt`;
- `sitemap.xml`;
- JSON-LD для Organization, Service и FAQPage.

При добавлении новых страниц обновлять sitemap и internal linking.

## Проверки перед сдачей изменений

Минимум:

```bash
npm run build
```

Для локального Docker:

```bash
docker compose up -d --build
docker compose ps
```

Визуально проверять:

- desktop 1440px;
- mobile 390px;
- hero;
- форму;
- карточки и длинные русские строки;
- отсутствие наложений;
- доступность CTA и интерактивных карточек с клавиатуры.

## Производительность

- Не добавлять тяжелые библиотеки ради простых эффектов.
- Изображения ниже первого экрана делать lazy, когда появится больше медиа.
- Анимации должны быть короткими и смысловыми.
- Не использовать видео/3D без понятной пользы для доверия и заявки.
